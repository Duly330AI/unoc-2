# 05. Real-time & UI Model

This document describes the real-time event system and the frontend architecture using React Flow.

## 1. Real-time Architecture

The application uses **Socket.io** to push state changes from the Node.js backend to the React frontend.

### 1.1 Connection

*   **Endpoint:** `/socket.io` (Standard Socket.io path).
*   **Transport:** WebSocket (preferred) with Polling fallback.
*   **Auth:** (Future) JWT token in handshake auth.

### 1.2 Event Inventory

Events are emitted when the server state changes. The frontend listens to these to update the Zustand store without polling.

| Event | Payload | Trigger | Description |
| :--- | :--- | :--- | :--- |
| `device:created` | `Device` object | `POST /devices` | A new device was added. |
| `device:updated` | `Device` object | `PATCH /devices/:id` | Attributes (name, position) changed. |
| `device:deleted` | `{ id: string }` | `DELETE /devices/:id` | A device was removed. |
| `link:created` | `Link` object | `POST /links` | A new link was added. |
| `link:deleted` | `{ id: string }` | `DELETE /links/:id` | A link was removed. |
| `status:updated` | `{ id: string, status: string }` | Status Service | Operational status changed (e.g., UP -> DOWN). |
| `signal:updated` | `{ id: string, rxPower: number }` | Optical Service | Optical signal levels changed. |

### 1.3 Coalescing

High-frequency updates (like signal calculations during a drag operation) are debounced on the server or coalesced into a single `batch:update` event if necessary (not in MVP).

## 2. UI Architecture

The frontend is a Single Page Application (SPA) built with **React** and **Vite**.

### 2.1 State Management (Zustand)

We use **Zustand** for global state.

```typescript
interface AppState {
  devices: Record<string, Device>;
  links: Record<string, Link>;
  selection: string[]; // IDs of selected items
  
  // Actions
  setDevices: (devices: Device[]) => void;
  addDevice: (device: Device) => void;
  updateDevice: (id: string, data: Partial<Device>) => void;
  // ...
}
```

### 2.2 Canvas (React Flow)

The network topology is visualized using **React Flow**.

*   **Nodes:** Custom Node components (`DeviceNode.tsx`) render the device icon, status indicator, and labels.
*   **Edges:** Custom Edge components (`LinkEdge.tsx`) render the connections, colored by status (Green=UP, Red=DOWN).
*   **Layout:** Manual positioning by default. Auto-layout (Dagre/Elk) can be triggered via a toolbar action.

### 2.3 Interaction Model

| Action | Interaction | Result |
| :--- | :--- | :--- |
| **Select** | Click Node/Edge | Updates `selection` in store. Shows details in Sidebar. |
| **Move** | Drag Node | Updates local state. Emits `device:updated` on drop. |
| **Link** | Drag from Handle to Handle | Creates a temporary connection line. Emits `POST /links` on release. |
| **Pan/Zoom** | Mouse Wheel / Drag Canvas | Updates Viewport transform. |
| **Context Menu** | Right-click Node | Shows actions (Delete, Properties, Provision). |

### 2.4 Panels

*   **Toolbar:** Tools for adding devices, toggling layers, and zoom controls.
*   **Sidebar (Properties):** Context-sensitive panel showing details of the selected object.
    *   *Device Selected:* Shows Name, Type, IP, Status, Optical Levels.
    *   *Link Selected:* Shows Source, Target, Distance, Attenuation.
    *   *Nothing Selected:* Shows Global Stats or Legend.

## 3. Error Handling

Standardized error codes are returned by the REST API and displayed as Toast notifications.

| Code | HTTP | Meaning |
| :--- | :--- | :--- |
| `DEVICE_NOT_FOUND` | 404 | ID does not exist. |
| `INVALID_LINK` | 400 | Incompatible endpoints (e.g., OLT to OLT). |
| `POOL_EXHAUSTED` | 409 | No IPs left in the pool. |
| `LOOP_DETECTED` | 400 | Link would create a cycle (if cycles are forbidden). |
| `INTERNAL_ERROR` | 500 | Unhandled backend exception. |

## 4. Determinism

*   **UUIDs:** Generated by the backend (or client-side v4 UUIDs) to ensure global uniqueness.
*   **Order:** Lists are typically sorted by `createdAt` or `name` to prevent UI jumping.
